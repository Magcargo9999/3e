### 문제 2. 소프트웨어의 개발 기간과 이용 기간 장기화에 의해 발생하는 문제점에 대해서 "추적 가능성"이 어떠한 효과가 있는지 설명하시오.

 * **개발 기간 장기화 시 문제점:** 개발 기간이 길어지면 소프트웨어의 개발흐름을 어떠한 기준으로 왜 그렇게 프로그래밍했는지 파악하기 어려워집니다. 특히 초기 개발자가 퇴사할 경우 코드의 핵심 개념과 히스토리를 아는 인력이 사라져 유지보수에 큰 어려움을 겪게 됩니다.
 * **이용 기간 장기화 시 문제점:** 장기 이용 시 반복적인 수정으로 인해 코드가 복잡해지며, 처음 설계 단계의 형태와 실제 운영 중인 프로그램의 형태가 크게 달라져 시스템 파악이 어려워집니다.
 * **추적 가능성(Traceability)의 효과:**  추적 가능성은 개발 과정에서 발생하는 결정 사항과 변경 사항을 문서화하여 요구사항부터 코드까지 연결하는 것입니다. 
    * 이를 통해 설계 도면과 히스토리를 명확히 남겨두면, 시간이 흐른 뒤에도 개발 의도를 파악할 수 있어 정보 유실 문제를 해결할수있습니다. 
    * 결과적으로 디버깅이나 코드 수정 시 영향 범위를 빠르게 파악하여 투입되는 ** 리소스(시간과 비용) **를 획기적으로 절감할 수 있습니다.
